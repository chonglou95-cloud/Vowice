# Vowice 问卷字段说明

## 研究框架

| 模块 | 要验证的假设 | 产品决策 |
|---|---|---|
| 使用现状 | 语音陪伴需求集中在独处、睡前和情绪波动场景 | MVP 首屏入口与情境模板 |
| 声音偏好 | 用户愿意调节声音，但更在意可感知特质而非专业参数 | 声契工坊的控件与试听方式 |
| 玩法选择 | 记忆、主动联系和情境通话中存在清晰优先级 | PRD P0/P1 排序 |
| 主动与记忆边界 | 用户希望被记住，但需要确认权和可撤回性 | 通知频率、记忆确认和删除机制 |
| 风险顾虑 | 隐私、依赖、越界、自然度会影响使用意愿 | 安全护栏和信任设计 |

## 字段字典

| 字段 | 类型 | 必填 | 含义 |
|---|---|---:|---|
| `ai_companion_usage` | 单选 | 是 | 近 3 个月 AI 对话/陪伴产品使用频率 |
| `companion_scenarios` | 多选，最多 3 项 | 是 | 需要陪伴的主要情境 |
| `preferred_interaction` | 单选 | 是 | 偏好的文字、语音消息或连续通话形式 |
| `voice_customization_interest` | 1–5 分 | 是 | 对自定义 AI 声音的兴趣程度 |
| `voice_traits` | 多选，最多 3 项 | 是 | 最想调节的声音特质，如温柔或清冷、语速和情绪表达 |
| `preferred_experiences` | 多选，最多 2 项 | 是 | 最有吸引力的产品玩法，包括“初声”等候选概念 |
| `proactive_contact` | 单选 | 是 | 可接受的主动联系方式 |
| `memory_content` | 多选，最多 3 项 | 是 | 可以被长期记住的信息 |
| `memory_control` | 单选 | 是 | 偏好的记忆写入控制方式 |
| `main_concerns` | 多选，最多 3 项 | 是 | 使用产品的主要顾虑 |
| `one_thing_wish` | 短文本 | 否 | 产品只能做好一件事时的期待 |
| `age_group` | 单选 | 是 | 年龄段，仅用于分组比较 |
| `gender` | 单选 | 否 | 性别认同，仅用于判断目标样本覆盖 |

## 建议分析方式

1. 先看各题频数，不在样本小于 100 时强调小数点百分比。
2. 交叉比较 `ai_companion_usage` 与 `preferred_experiences`，判断深度用户和新用户的差异。
3. 交叉比较 `voice_customization_interest` 与 `voice_traits`，决定调音功能深度。
4. 把 `proactive_contact`、`memory_control`、`main_concerns` 一起看，形成信任设计原则。
5. 对开放题做主题编码，并保留与团队原假设相反的回答。

## 决策阈值建议

- 某项玩法进入 P0：在目标样本中进入前两名，且开放题中能找到具体使用场景。
- 某项玩法暂缓：选择率低，或只有概念兴趣、没有真实行为证据。
- 某项风险成为护栏：进入顾虑前三，或开放题中出现明确的高严重度越界担忧。

这些阈值用于首轮方向判断，不代表统计显著性结论。
