# Vowice 原型范围与页面清单

## 1. 原型目标

原型用来验证三个关键问题：

1. 用户能否理解“塑造声音”，并在不看教程的情况下完成。
2. 连续通话、情境声场和状态反馈能否产生比语音消息更强的沉浸感。
3. 用户是否理解“候选记忆需要我确认”，并信任这套机制。

## 2. 信息架构

### 主导航

- **今日**：主动声音、继续上次话题、快速进入情境。
- **通话**：启动或继续语音陪伴。
- **记忆**：共同记忆、关系时刻和回声日记。
- **声契**：角色、声音配方、关系边界和联系频率。

## 3. P0 页面

| ID | 页面 | 关键状态/内容 |
|---|---|---|
| O-01 | 成年确认与 AI 身份提示 | 18+、AI 交互对象、麦克风和数据用途摘要 |
| O-02 | 创建关系 | 名字、称呼、关系阶段、沟通方式 |
| V-01 | 选择基础音色 | 3–6 把授权声音、即时试听 |
| V-02 | 调整声音配方 | 温度、距离感、克制、活力、语速、气息感 |
| V-03 | 声音确认 | 完整试听、保存、重新调整 |
| H-01 | 今日 | 最新声音胶囊、今日问候、快速情境入口 |
| C-01 | 情境选择 | 随便聊聊、想说件事、深夜放松、睡前陪伴 |
| C-02 | 通话中 | 倾听/理解/说话、字幕开关、声场开关、结束、打断/重说 |
| C-03 | 回复修复菜单 | 太说教、不像他、越界了、重复了 |

| S-01 | 声契设置 | 声音、边界、主动频率、记忆管理、删除数据 |

## 4. 必须设计的异常状态

- 麦克风权限被拒绝。
- 语音过短或无法识别。
- 网络断开、模型超时或语音生成失败。
- 回复正在生成时用户退出。
- 播放被电话或其他媒体中断。
- 记忆候选为空。
- 用户删除了正在被主动来信引用的记忆。
- 检测到严重越界、自伤风险或不适宜内容。

## 5. 交互原型连接要求

- 主流程从 O-01 连接到 L-01，可在 5–7 分钟内完整演示。
- 可用性测试需要的任务必须可点击，不用主持人口述“这里本来可以”。
- 所有按钮、输入、加载和错误状态与 MVP 中的名称一致。
- 使用真实中文文案和经授权/合成的音频样本，不使用“占位文本”做测试。

## 6. 视觉方向

- 这是一个成人使用的私密声音工具，不做幼态可爱、粉色满屏或恋爱游戏式的数值面板。
- 声音是第一信息：波形、时间、播放状态和环境声层级需清楚，但避免把波形做成纯装饰。
- 声音控件使用滑杆、分段控件、开关和 A/B 试听，不把所有决策放进文字表单。
- 安全区域、字体和触控区域以 iPhone 为主要设备设计，同时保证 Android 不溢出。
