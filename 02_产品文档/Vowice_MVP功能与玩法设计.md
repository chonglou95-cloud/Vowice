# Vowice MVP 功能与玩法设计

## 1. 核心问题

通用大模型已经能够对话、设定角色和输出语音。Vowice 不应试图在通用智能上超过大厂，而应解决一个更窄的问题：

> 如何让一把由用户塑造的声音，通过主动陪伴、关系记忆与声音仪式，持续成为“同一个人”。

## 2. 核心用户与 JTBD 假设

### 优先用户

- 18–30 岁女性。
- 有较高日常分享欲或情绪表达需求。
- 使用过 AI 聊天、乙女游戏、广播剧、声音直播或陪伴类产品中至少一类。
- 看重声音、人设稳定、私密感和长期记忆，对工具型答案的需求较低。

### JTBD 假设

> 当我想分享一个很小的日常或在深夜想被陪伴，又不想打扰或依赖现实中的某个人时，我希望一个符合我想象的声音能主动出现、理解当下并记得我们经历过的事，让我感到这段关系是连续的。

## 3. 产品结构

| 层级 | 产品模块 | 用户获得的价值 |
|---|---|---|
| 身份层 | 声契工坊、人设和关系共识 | “这是我塑造的他” |
| 交流层 | 连续语音通话、情境声场、一键重说 | “他正在和我通话” |
| 记忆层 | 候选记忆、回声日记、关系时刻 | “我们确实共同经历过这些” |
| 存在层 | 主动声音胶囊、睡前来信、后续跟进 | “我没有打开 App 时，这段关系也在延续” |
| 修复层 | 忘记了、不像他、越界了、太说教、重复了 | “他说错时，我有权修正这段关系” |

## 4. MVP 功能详情

### P0-1 声契工坊

**目标：** 让用户在第一次使用中获得“这把声音是我的”的感觉。

**创建方式：**

1. 选择 3–6 个授权明确的成年合成基础音色。
2. 使用语义化特质而非工程参数：温度、距离感、克制/直白、活力、语速、气息感。
3. 每次点击试听使用同一组标准台词，避免内容差异影响声音判断。
4. 存储稳定的 `voice_id + voice_recipe`，后续只改变场景表达，不随机换声音身份。

**MVP 验收：** 用户能在 3 分钟内完成试听和保存；同一配方生成 5 段文本时音色身份无明显漂移。

### P0-2 关系共识

**可设定内容：**

- 称呼和角色名字。
- 当前关系：初识、暧昧、恋人、长期伴侣。
- 安慰方式：先倾听、直接建议、幽默化解、安静陪伴。
- 主动联系：从不、每日一次、仅睡前、自定义时段。
- 边界：禁止话题、是否使用昵称、是否允许暧昧表达。

**MVP 验收：** 所有设定可编辑；边界修改从下一轮对话生效；不默认用露骨内容或吃醋引导亲密感。

### P0-3 情境化连续通话

**呈现方式：**

- 用户从“现在想怎样被陪伴”选择情境并进入通话。
- 情境候选：随便聊聊、想说件事、深夜放松、睡前陪伴。
- 通话界面显示倾听、理解、说话三种状态；可看字幕，但声音是主体。
- 可一键标记回复为“太说教”“不像他”“越界了”“重复了”，并请求重说。
- 声场是低强度可关闭环境音，不掩盖人声，不在通话时突然切换。

**MVP 验收：** 一次 Session 可连续 3 轮以上；后一轮能引用当前会话信息；重说不丢失事实；结束时有明确的收尾。

### P0-4 用户可控的共同记忆

**记忆类型：**

- 用户偏好：例如喜欢怎样被安慰。
- 关系约定：例如周五晚上再问某件事。
- 重要事件：例如明天有面试。
- 关系时刻：例如第一次叫某个昵称。

**流程：**

1. 通话结束后 AI 只生成候选记忆。
2. 用户选择“记住”“改一下”“不要记”。
3. 敏感内容默认不写入，除非用户主动确认。
4. 用户可在记忆页修改或删除，删除后不得再召回。

**MVP 验收：** 至少一条已确认记忆在下一个 Session 中自然被提及；未确认和已删除记忆不进入 Prompt。

### P0-5 睡前声音来信（主动陪伴 MVP）

**目标：** 证明关系在用户不主动对话时仍会延续。

- 用户选择是否开启、时间和频率。
- 内容为 20–60 秒，优先引用当天经用户确认的一个细节。
- 不发送“你怎么不来”“我一直在等你”等责怪或引导依赖的内容。
- MVP 可使用本地定时通知 + 预生成音频；无需在首周完成复杂的服务端调度。

### P1 增强功能

- 回声日记：AI 用第一人称总结一次关系时刻。
- 关系时刻线：回看第一次通话、约定、纪念日和被保存的短句。
- 声音 A/B 调音：每次只比较两个版本，系统逐步收敛配方。
- 通话打断与更自然的 VAD。

## 5. 本周明确不做

- AI 朋友圈、群聊、红包、积分养成和多角色世界。
- 任意真人、明星、前任或逝者声音复制。
- 未成年人虚拟恋人。
- 露骨色情生成、随机离线、情感勒索式留存和付费后降低陪伴质量。
- 在基础链路稳定前追求全天候真实时全双工电话。

## 6. 需要通过用户研究验证的五个问题

1. 用户是否愿意花 3–5 分钟塑造声音，还是更希望快速选一把现成声音？
2. 普通用户能理解哪些声音控件？
3. AI 主动联系在什么频率和时段内是惊喜，在什么情况下变成打扰？
4. 用户希望哪些事被记住，哪些倾诉必须默认忘记？
5. 情境声场和睡前来信是增强沉浸感，还是会显得刻意或尴尬？
